# 디자인 의도 규칙 (2026-07)

## 배경

P1~P8 디자인 개편 이후 접수된 사용자 피드백 5건에 대한 대응이자, 이후의 모든 UI 변경이
"자의적 재배치"가 아니라 **규칙의 적용**이 되도록 판단 기준을 명문화한 문서다.

| # | 피드백 | 대응 |
|---|--------|------|
| 1 | 홈탭 배열이 중요도와 불일치, 버튼 가시성 부족 | 규칙 1(정보 위계)·규칙 2(강조 위계)로 재배치 |
| 2 | 보충 진입 버튼이 내보내기처럼 생긴 아이콘뿐 | 규칙 2 — 글리프 교체(edit_note) + 아이콘·라벨 버튼화 |
| 3 | 보충탭에서 캐릭터 정보가 화면 절반뿐 | 규칙 4 — 컨트롤 기본 접힘, 콘텐츠 우선 |
| 4 | 탭 위치는 저장되는데 정렬·필터는 저장 안 됨 | 규칙 3 — 목록 표면 상태 저장 완결 |
| 5 | 개선인지 단순 재배열인지 의도를 모르겠음 | 본 문서 — 변경마다 의도 문장, 규칙 명문화 |

CLAUDE.md의 5대 원칙(특히 원칙 02 실질적 기능성, 원칙 04 조작 마찰 최소화)이 상위 기준이며,
이 문서는 그 원칙을 화면 배치·강조·상태 저장에 적용하는 실무 규칙이다.

---

## 규칙 1 — 홈 정보 위계

홈 대시보드는 위에서 아래로 **"지금 해야 할 일" → "가끔 쓰는 도구"** 순으로 읽힌다.

| 계층 | 내용 | 현재 요소 |
|------|------|-----------|
| ① 복구 필요 오류 | 사용자의 행동을 촉구하는 문제 | 어시스턴트 정합성 오류 카드(조건부) |
| ② 이어서 할 작업 | 세션마다 바뀌는 동적·개인적 맥락 | 최근 활동(조건부) |
| ③ 시점 민감 정보 | 오늘/이번 주에만 의미 있는 정보 | 다가오는 생일(조건부) |
| ④ 생성 진입 | 새 작업의 시작점 | 바로가기(캐릭터 추가·사건 추가·검색) |
| ⑤ 저빈도 도구 | 세션당 0~1회 쓰는 이동 타일 | 도구(연표·이름 은행·이미지·엑셀) |

- 동적 정보(②③)가 고정 버튼(④)보다 위에 있는 이유: 고정 버튼은 항상 같은 자리라
  근육기억으로 찾지만, 동적 정보는 노출되지 않으면 존재를 모른다(원칙 04:
  "일일이 확인하지 않으면 존재를 알 수 없는 데이터가 있어서는 안 된다").
- 엑셀 내보내기·가져오기가 ⑤에 있는 이유: 개발 의도가 핵심 기능으로 명시한 엑셀 왕복을
  설정 3뎁스에서 홈 1탭으로 끌어올렸다. 실행 로직은 `ExcelTransferController`로 추출해
  설정 화면과 공유한다(중복 금지).
- **신규 홈 요소는 먼저 이 5계층 중 하나에 배정한 뒤, 그 계층 안에서 배치한다.**
- 각 섹션은 자기완결 블록(레이아웃 컨테이너 1개) — 순서 변경은 블록 스왑만으로 가능해야 한다.

## 규칙 2 — 강조 위계

시각 강조가 곧 기능 위계다. 스타일은 디자인 시스템(styles.xml)의 것만 쓴다.

- **채움 버튼(`Widget.App.Button`)은 화면당 1개** — 그 화면의 핵심 행위에만.
  홈에서는 '캐릭터 추가'. (알려진 예외: 랜덤 보충 탭 편집 모드에서 다시 뽑기·저장이
  동시에 채움 — 기존 상태, 후속 정리 대상.)
- 보조 액션은 `Widget.App.Button.Tonal`, 중립/취소는 `.Outlined`, 다이얼로그/링크는 `.Text`.
- 카드 톤: 콘텐츠 카드 = 외곽선(`Widget.App.Card`), 이동/통계 타일 = `Widget.App.Card.Tonal`.
  홈 도구 카드는 "이동 타일"이므로 토널.
- **오독 여지가 있는 진입점은 아이콘 단독 금지** — 아이콘+텍스트 라벨로 노출한다.
  적용 사례: 캐릭터 탭의 보충 진입(구 upload-file 글리프 → edit_note + "보충" 라벨 버튼,
  `character_menu.xml`의 `actionLayout`).
- 글리프는 의미가 기능과 일치해야 한다. 파일에 화살표가 든 글리프(업로드/다운로드 형)는
  가져오기·내보내기 계열에만 쓴다.

## 규칙 3 — 목록 표면 상태 저장

**정렬·필터·접힘·탭 위치는 사용자가 만든 작업 컨텍스트다 — 재방문 시 그대로 복원한다.**

- 저장 매체: 화면별 SharedPreferences `*_ui_state` 파일. 검색어만 예외 —
  일회성 조회이므로 세션(SavedStateHandle) 한정. 낡은 검색어가 재방문 시 목록을
  조용히 줄이는 것을 막는다(변수 제어).
- 복원값 위생: 삭제된 엔티티를 가리키는 복원값은 해제한다. 보이는 컨트롤이 없는 필터
  (예: 기준 설정에서 꺼진 이슈의 저장된 필터)는 유지하지 않는다 — 무음 필터 금지.
  필드는 인덱스가 아니라 안정 키로 저장·복원한다(목록이 스코프별로 달라지므로).
- **전수 원칙:** 정렬·필터·세그먼트 컨트롤이 있는 화면은 예외 없이 이 규칙을 따른다.
  아래 표가 전 화면 현황이며, 새 컨트롤은 여기에 등재한다. (2026-07 전수 조사 기준.)
- 설정→앱 초기화는 아래 표의 모든 파일을 지운다(테마 제외).

| prefs 파일 | 화면 | 저장 항목 |
|------------|------|-----------|
| `character_list_ui` | 캐릭터 목록 | sort_kind/field_key/ascending/body_part, field·tag·novel_filters_json, birthday_banner_collapsed |
| `analysis_ui_state` | 분석 허브 | last_tab |
| `supplement_ui_state` | 보충 | universe_id, novel_id, sort_mode, random_mode, last_tab, issue_filter, filter_collapsed, random_controls_collapsed |
| `image_manager_ui_state` | 이미지 관리 | filter_base, filter_tags(JSON), sort |
| `search_ui_state` | 통합 검색 | sort_mode, field_filters_json |
| `timeline_ui_state` | 연표 | filter_novel_id, filter_character_id, zoom_level, center_year |
| `graph_ui_state` | 관계도 | universe_id, novel_id, rel_types_json, faction_ids_json, 표시 토글 |
| `stats_prefs` | 통계 | selected_novel_id, pattern_insights_enabled_types, ranking_ascending/universe_id/novel_id/field_key/body_part |
| `namebank_ui_state` | 이름 은행 | show_only_available |
| `assistant_prefs` | 어시스턴트 | 카테고리 on/off 필터 |
| `supplement_criteria` | 보충 완성도 검사 | 검사 기준 토글 |
| `field_manage_ui_state` | 필드 관리 | entity_type(캐릭터/사건 세그먼트) |

**검색어(query)는 위 어디에도 저장하지 않는다** — 세션 한정(캐릭터 목록·통합 검색·
연표·이름 은행은 VM 필드, 이미지 관리는 SavedStateHandle). 재방문 시 빈 검색어로 시작한다.

## 규칙 4 — 보충탭 정보 우선

**콘텐츠(캐릭터 정보)가 컨트롤보다 먼저다.**

- 컨트롤(필터·뽑기 옵션)은 기본 접힘. 접힌 헤더의 **한 줄 요약**이 현재 조건을 항상
  보여준다 — "일일이 열어봐야 아는 상태"를 만들지 않는다(원칙 04).
- 핵심 루프는 접힘과 무관하게 1탭 유지: ◀ 다시 뽑기 ▶ 행과 인라인 편집 스위치는 상시
  노출. 편집 스위치는 '지금 편집 중'임을 알리는 유일한 상시 표지이기도 하다.
- 접기/펼치기 조작은 데이터 상태를 바꾸지 않으므로 미저장 편집 가드와 무관하게 동작한다.
- 접힘 상태는 규칙 3에 따라 영속된다.

## 이 라운드 적용 내역 (요약)

- 홈: 섹션 순서 오류→최근→생일→바로가기→도구, 캐릭터 추가 채움 버튼화, 도구 카드
  토널 타일화, 엑셀 카드 신설(`ExcelTransferController` 추출).
- 보충 진입: `ic_supplement` 글리프 edit_note로 교체 + `action_supplement.xml` 라벨 버튼.
- 보충탭: 필터 카드·뽑기 옵션 기본 접힘 + 한 줄 요약, 내부 탭 위치·이슈 필터 영속.
- 이미지 관리: 기본 필터·태그·정렬 영속(검색어는 세션 한정).
- 설정 앱 초기화: prefs 목록의 죽은 이름 4개 교정 + 누락 파일 5개 추가.
