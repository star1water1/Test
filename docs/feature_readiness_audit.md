# 기능 점검 및 개선안 설계 리뷰 (코드 수정 없음)

## 1) 결론 요약
- 최근 설계 문서의 5대 요구사항 중 **핵심 골격은 대부분 이미 반영**되어 있습니다.
- 다만 사용자 경험/운영 안정성 관점에서 아직 보완이 필요한 항목이 있어, 실제 릴리스 전에는 아래의 P0~P2 개선안을 권장합니다.

### 현재 준비도 (요약)
- 카드 재정렬: **부분 완료 (핵심 동작 구현, UX 충돌 일부 잔존)**
- 이미지 확대 보기: **완료 (전체화면 + 확대/이동 + 인덱스)**
- anotherName(이명): **완료 (DB/입력/상세/검색/엑셀 연동)**
- 설정 탭: **부분 완료 (탭/테마/버전 정보 중심, 시스템 관리 기능 미구현)**
- 다크모드: **완료에 근접 (DataStore 저장 + 즉시 적용), 초기 적용 타이밍 개선 여지 존재**

## 2) 요구사항별 상세 점검

## 2-A) 카드 드래그 재정렬

### 확인된 구현
- Universe/Novel/Character 모두 `displayOrder` 필드와 정렬 쿼리를 사용합니다.
- 세 리스트에 `ItemTouchHelper` 기반 순서 변경 저장 로직이 존재합니다.

### 잔여 갭 / 리스크
1. **편집/삭제 진입이 여전히 롱프레스 중심**
   - 설계 의도(롱프레스 제거, `⋮` 메뉴 통일)와 불일치.
2. **캐릭터 리스트는 드래그 핸들 없이 카드 롱프레스 드래그**
   - Universe/Novel과 UX 일관성이 다르고, 비교모드/롱프레스 편집과 충돌 여지.
3. **검색/필터 상태에서 재정렬 제한 정책 부재**
   - 현재 보이는 부분집합만 재정렬 시 원본 전체 순서의 의도치 않은 왜곡 가능.
4. **신규 insert 시 `max(displayOrder)+1` 보장 로직 부재**
   - 기본값 0으로 들어가면 기존 항목과 순서 충돌 가능.

### 개선 권장
- P0: 편집/삭제 진입을 카드 우측 `⋮` 메뉴로 통일.
- P0: 캐릭터 카드에도 핸들(≡) 도입 + `startDrag` 패턴 통일.
- P1: 검색어/필터 활성 시 재정렬 비활성화(툴팁 제공).
- P1: Repository insert 시 다음 순번 자동 할당 헬퍼 추가.

## 2-B) 사진 탭 시 확대 보기

### 확인된 구현
- 상세 화면에서 이미지 탭 시 전용 Viewer Fragment로 이동.
- Viewer에 ViewPager + ZoomableImageView(핀치/더블탭/팬) 적용.
- 파일 canonical path 검증으로 앱 내부 경로만 로드.

### 개선 권장
- P1: 닫기 버튼의 접근성 텍스트/포커스 순서 QA 점검.
- P2: 초고해상도 이미지 대응이 필요하면 `SubsamplingScaleImageView` 대체 검토.

---

## 2-C) anotherName(이명) 기본 필드

### 확인된 구현
- Room 엔티티/마이그레이션(v6→v7) 반영 완료.
- 편집 화면 입력 필드, 상세 화면 표시(빈값 숨김) 반영.
- 검색은 `name OR anotherName`으로 확장.
- Excel Spec/Export/Import 모두 `이명` 컬럼 처리(구버전 미존재 열 호환).

### 개선 권장
- P2: 캐릭터 카드에도 선택적으로 이명 서브타이틀 노출 옵션 추가(설정화).

---

## 2-D) 설정 탭 + 인앱 시스템 관리

### 확인된 구현
- 하단 탭/네비게이션에 `settingsFragment` 연결 완료.
- 현재 설정 화면은 사실상 **테마 + 버전 정보** 중심의 최소 형태.

### 미구현 영역 (설계 대비)
- 데이터 섹션(Excel 가져오기/내보내기 진입)
- 관리 섹션(무결성 점검, displayOrder 재색인, 캐시 정리)
- 진단 결과 표시 및 복구 플로우

### 개선 권장
- P0: 설정 탭에 데이터 관리 진입점(Import/Export)을 우선 배치.
- P1: `SystemMaintenanceService` 도입:
  - orphan/fk 이상 카운트
  - displayOrder 중복/누락 검출 + 재색인
  - 끊어진 이미지 파일 경로 검출
- P1: 점검 결과를 "읽기 전용 리스트 + 항목별 복구 버튼"으로 제공.

## 2-E) 다크모드

### 확인된 구현
- DataStore에 `theme_mode` 저장.
- MainActivity 시작 시 저장된 테마를 읽어 `AppCompatDelegate` 적용.
- 설정 화면에서 즉시 변경 가능.

### 개선 권장
- P1: 앱 프로세스 시작 시점(Application) 선적용 검토(초기 깜빡임 최소화).
- P2: 주요 화면 명도 대비(카드 배경/보조텍스트/칩) 체크리스트 정식화.

## 3) 추가로 제안하는 기능 (요구사항 외)

### A. 재정렬 운영성 강화
1. **정렬 잠금 토글**: 실수 드래그 방지.
2. **정렬 되돌리기(Undo 1회)**: 사용자 신뢰도 개선.
3. **재색인 자동 실행 트리거**: import 직후/대량 삭제 후.

### B. 검색 품질 향상
1. 초성/자모 검색 옵션(한글 UX 개선).
2. `name`, `anotherName`, `tags` 가중치 기반 정렬(정확도 향상).
3. 고급 필터(작품/태그/이미지 보유/메모 유무).

### C. 유지보수/안정성
1. Room Migration 테스트 추가(5→6→7 연속).
2. Excel round-trip 골든 파일 테스트(이명/코드/정렬순서 포함).
3. Viewer 메모리 회귀 테스트(연속 스와이프, 회전, 백그라운드 복귀).

### D. 사용자 가치가 큰 확장
1. **설정 탭에 백업 상태 카드**(마지막 성공 시각, 최근 오류).
2. **캐릭터 중복 감지**(이름 유사도 + 동일 작품).
3. **관계망 시각화(그래프 뷰)**(중장기).

- 검색 고도화(가중치/고급필터)
