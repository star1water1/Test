## P0 — 치명 (3건)

### P0-1. 이미지 포함 백업이 커지면 복원이 불가능해진다
- **시나리오:** 이미지 수백 장 헤비유저의 자동 백업(.enc)을 복원 (대하 장편 작가 #5)
- **문제:** 자동 백업은 크기 제한 없이 전체 이미지를 포함해 생성되지만, 복원 경로는 `importFromLocalFile()`의 **50MB 상한**을 통과해야 한다. 백업은 매일 "성공"하는데 정작 복원 시점에 "파일이 너무 큽니다"로 거부 — 기기 이전·데이터 사고 시 사실상 전체 유실.
- **근거:** `AutoBackupWorker.kt:83-118`(상한 없는 생성), `ExcelImporter.kt:98-106`(50MB 거부), `MAX_IMPORT_FILE_SIZE`(1059행)
- **위반 기준:** ①②④

### P0-2. 엑셀 필수 컬럼 삭제/이동 시 무통보 실패 + 이웃 컬럼 오기록
- **시나리오:** 필수 컬럼을 실수로 삭제한 파일 들이기 (엑셀 파워유저 #4)
- **문제:** (a) 첫 컬럼(이름/제목/연도)이 삭제·이동되면 시트 전체가 **오류·경고 0건으로 조용히 스킵**된다. 안내 시트는 "헤더 순서 변경 가능"이라고 약속하는데 첫 컬럼 이동만으로 전체 무시. OVERWRITE 전략과 결합하면 기존 데이터 삭제만 실행됨. (b) 첫 컬럼이 아닌 필수 컬럼 삭제 시 **하드코딩 인덱스 폴백**(`cols["사건 설명"] ?: 4`)이 이웃 컬럼 데이터를 그 자리로 읽어 잘못된 값이 조용히 DB에 기록된다.
- **근거:** `ExcelImportService.kt:1031-1037, 1056, 1182, 1316, 1413, 1650, 1793`, `ExcelExporter.kt:424,435`
- **위반 기준:** ②

### P0-3. 암호화 백업(.enc)이 기기 종속 — 폰 교체 시 복원 불가, 안내 전무
- **시나리오:** 폰 교체 시 데이터 이전 (입문자 #5)
- **문제:** '백업 내보내기'의 .enc 파일은 AndroidKeyStore **기기 종속 키**로 암호화되어 새 폰에서 절대 복호화할 수 없다. `allowBackup=false`라 OS 이전도 차단. 실제 이전 수단은 엑셀 내보내기뿐인데 UI 어디에도 "이 파일은 이 기기 전용", "기기 이전은 엑셀 내보내기 사용" 안내가 없다. 구폰 초기화 후 .enc만 있으면 영구 유실.
- **근거:** `BackupEncryptor.kt:18-57,231-238`, `SettingsFragment.kt:457-474`, `AndroidManifest.xml:14`
- **위반 기준:** ②④
