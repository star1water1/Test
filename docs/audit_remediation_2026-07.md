# 최근 기능 감사 후속 수정 설계서 (2026-07) — PR-F1~F3

> 배경: "최근 업데이트 기능이 의도대로 동작하는가" 감사(4영역 병렬) 결과, 검색/필터/정렬은 의도대로였고 일괄편집·엑셀·이미지에서 갭이 확인됨. 본 문서는 확인된 갭 전부를 **우선순위 순서**로 수정하는 계획이다.
> 판단 기준: CLAUDE.md 개발 의도 — ① 받쳐주는 확장성, ② **변수 제어(데이터가 말없이 유실·왜곡되지 않는다: 검증→알림→교정경로)**, ③ 자율성, ④ 엑셀 왕복 무결성. 방편식 패치 금지.

## 수정 원칙

- 각 수정은 **의도된 동작을 완결된 형태로** 구현한다(임시 가드/특정 케이스 금지).
- 무음 유실(우선순위 1)은 각 항목을 **구현 전 코드로 재확인**한 뒤 고친다.
- 각 PR은 master 대상, CI(빌드+단위테스트) 그린 후 병합. 로컬 SDK 부재 → 정적 리뷰 + CI가 검증.

## PR-F1 — 무음 유실 3건 (최우선, 대원칙 위배)

### F1-1. 엑셀: 연표/상태변화/관계/관계변화 시트의 "열 없음 = 유지" 미적용
- **의도(④/②):** 열이 **없으면** 기존 값 유지, 셀이 **빈칸**이면 삭제/무값(+요약). 캐릭터·세계관·작품·세력 시트는 이미 `X: T? = if (colIndex>=0) getCell(...) else null` + `?: existing` 패턴으로 올바름.
- **현상:** 아래 시트의 **선택 속성 열**은 위치 폴백(`cols["월"] ?: 1`)으로 인덱스를 잡고 `copy(month = month, …)`로 **무조건 덮어쓴다**. 열을 삭제하면 이웃 열을 오독하고, 값이 조용히 null/왜곡됨.
  - 연표(timeline): 월·일·역법 (`ExcelImportService.kt:1890–1892, 1997–1998`)
  - 상태변화(state change): 월·일·설명 (`~2145–2149, ~2223–2224`)
  - 관계변화(relationship change): 관계유형·설명·강도·양방향 (`~2400–2403, ~2498–2505`)
  - 관계(relationship): 설명 (`~2270, ~2357`)
- **방침:** 각 선택 속성 열에 대해
  1. 위치 폴백 제거 → `val monthColIndex = cols["월"] ?: -1` (필수 열인 연도·캐릭터·연결키는 현행 유지).
  2. copy()에서 **열-present 가드**: `month = if (monthColIndex >= 0) month else existing.month` (연표의 `eventType/isTemporary/createdAt`가 이미 쓰는 패턴과 동일).
  3. 열은 present인데 셀이 빈칸이면 현행 파싱 결과(null/삭제)를 따르되, **삭제성 변화는 clearedFields 요약에 집계**(F2-3과 동일 채널).
- **완결성 체크:** 위치 폴백을 쓰던 다른 시트도 동일 점검(같은 함수 내 required vs optional 구분). "사건 설명"은 이미 폴백 제거됨(주석 존재) — 같은 방식 확장.

### F1-2. 이미지: 삭제가 비트랜잭션 — 파일 삭제 실패 시 캐릭터가 링크를 조용히 잃음
- **의도(②):** 삭제 실패 시 데이터 링크가 온전해야 하고, 다중 owner 갱신은 원자적이어야 한다.
- **현상:** `ImageManagerViewModel.kt:178–198`가 DB `imagePaths`를 **먼저** 제거하고 그다음 `file.delete()`. delete가 false여도 링크는 이미 끊김. 여러 owner를 순차 갱신하다 예외 시 부분 유실.
- **방침:**
  1. 순서 역전: **파일 삭제를 먼저** 시도 → 성공 시에만 DB 링크 제거. 또는 링크 제거+파일삭제를 하나의 논리 단위로 하고 파일삭제 실패 시 DB 롤백.
  2. 다중 owner imagePaths 갱신은 **단일 트랜잭션**(`withTransaction`)으로 묶어 all-or-nothing.
  3. 파일이 이미 없으면(고아) 링크만 정리하는 것은 정상 — 이 경우는 성공으로 처리.
  4. 실패 건수는 기존 `BulkDeleteResult.failed` 채널로 정확히 보고(이미 존재).

### F1-3. 이미지: 일괄 삭제가 필터 화면 밖 선택분까지 삭제
- **의도(②/⑤):** 일괄 삭제는 사용자가 보는(필터된) 대상에만 작용해야 하고, 확인은 대상을 특정할 수 있어야 한다.
- **현상:** `ImageManagerFragment.kt:175–178` `selectedItems()`가 필터된 `currentList`가 아니라 **전체 `viewModel.images.value`**에서 선택 필터. 선택은 필터 전환 시 유지되므로 화면 밖 항목이 삭제됨.
- **방침(택1, 구현 시 확정):**
  - (A) 필터 전환 시 **보이지 않게 된 선택을 자동 해제**(선택은 현재 뷰 기준) — 가장 직관적, 조작 마찰 최소.
  - (B) `selectedItems()`를 현재 표시 목록(`currentList`) 교집합으로 한정 + 확인 다이얼로그에 "표시 중 N건만 삭제" 명시.
  - 권장 (A): 선택 모델을 "현재 뷰에서 보이는 것"으로 일관. 다만 정렬 변경(같은 집합)에서는 선택 유지. 필터(집합 변경) 전환 시에만 사라진 항목 선택 해제.

## PR-F2 — 고지/교정경로 미완 4건 (손실 없음, 변수 제어 마지막 단계)

### F2-1. 일괄편집: 작품 이동 복합 결과가 필드 삭제 경고를 가림
- **의도(②):** 동기화 실패와 필드/세력 제거가 동시에 나면 **둘 다** 고지.
- **현상:** `CharacterListFragment.kt:511–555` 결과 핸들러가 상호배타(`if syncFailures … else if move.hasRemoval …`). 동시 발생 시 제거 경고 억제.
- **방침:** 상호배타 분기를 **합산 메시지**로 변경 — sync 실패 N건 + 제거 M건(휴지통 보관)을 한 스낵바/다이얼로그에 함께 표기. OpResult detail에 두 카운트 모두 포함.

### F2-2. 일괄편집: 다른 세계관 캐릭터가 필드값 일괄에서 제외되지만 결과 미집계
- **의도(②):** 제외(스킵)는 **집계+고지**.
- **현상:** `BatchEditViewModel.kt:170–188` `scoped = ids.filter { universe == fieldDef.universeId }` 후 드롭 수(`ids.size - scoped.size`)를 `skipped`에 미반영. novelId=null 캐릭터도 조용히 드롭. (사전 배너 경고는 있음.)
- **방침:** 상태변화 중복 스킵 경로(`batch_success_with_skipped`)와 **대칭**으로, 세계관 불일치·미배정 드롭 수를 `skipped`로 집계해 결과에 "N명 제외(다른 세계관/미배정)"로 표기.

### F2-3. 엑셀: 빈칸 삭제(메모/이명/설명 등)가 "지워진 값" 요약에 미포함
- **의도(④/②):** 빈칸=삭제 + **요약 고지**.
- **현상:** `clearedFields` 집계가 이미지·태그·동적필드·이벤트링크에만 증가. 메모/이명/성/이름/세계관·작품·세력 설명의 빈칸 덮어쓰기(`""`)는 미집계·무경고.
- **방침:** 이들 필드의 (기존값 non-blank → 임포트 빈칸) 전이를 `clearedFields`에 집계(엔티티·필드명 라벨). 요약 다이얼로그(`ExcelImporter.kt:1029`)에 자연히 노출.

### F2-4. 이미지: 재압축 원본 되돌리기(휴지통 백업) — 백로그 #61 구현
- **의도(②/앱 일관성):** 재압축은 화질 저하·원본 삭제를 고지(현행 O)하되, 다른 삭제처럼 **되돌릴 수 있어야** 한다.
- **현상:** `ImageManagerViewModel.kt:367–376` 커밋 3단계가 원본을 영구 삭제, 복원 경로 없음.
- **방침(결정):** 커밋 시 원본 파일을 **영구 삭제하지 않고** `recompress_backup/` 디렉터리로 이동(경로 매핑 기록). 
  - 즉시: "실행취소" 스낵바 액션 → 백업본을 원위치 복원 + DB 경로 되돌림(세션 undo).
  - 지속: 백업본은 `SystemMaintenanceService` 고아 정리 대상에서 **제외**하고 **N일(기본 7일) 후 자동 정리**(용량 압박 시 우선). StorageAnalyzer 분류에 "재압축 백업"으로 표기.
  - 트랜잭션: 이동 실패 시 재압축 커밋 중단(원본 보존). 복원은 파일 복귀 후 DB 갱신.

## PR-F3 — 성능·경미

### F3-1. 이미지 어댑터: 선택 토글마다 전체 재바인드 + 썸네일 캐시 부재
- **현상:** `ImageManagerAdapter.kt:33–37`가 `notifyDataSetChanged`, bind()가 매번 디스크 재디코드(LruCache 없음).
- **방침:** 선택 상태 변경을 `notifyItemChanged(pos, payload)`로 부분 갱신, payload면 이미지 재디코드 생략. 어댑터에 CharacterAdapter식 `LruCache<String,Bitmap>` 썸네일 캐시 도입(픽셀 상한 유지).

### F3-2. 엑셀: numeric-code 방어 2/8 열만 · name/title 절단 미가드 · 관계 링크/세력 매칭
- **방침:** `getCellCode`(NUMERIC 강제→경고)를 세계관·작품·세력·사건·상태변화·관계변화 코드 열에도 적용. `name/title`도 `setTextSafe`(32,767 절단+per-value 경고)로 내보내 초장문에서 전체 실패 방지. 관계변화 `event.code` null 레거시는 자연키(캐릭터+연도) 폴백으로 링크 보존; 관계 세력 name-only 매칭에 코드 열 추가 검토.

### F3-3. 일괄편집 잔여
- batch 삭제 확인에 cascade 요약(`getBatchDeleteImpact`: 관계/상태변화/세력/이미지 N건) 추가(단건과 대칭). `changeNovel` 시맨틱 재싱크의 `standardYear` 게이트를 BIRTH/DEATH/ALIVE 역할에는 불필요하게(역할별 분기).

### F3-4. 필터 UX·의미
- **CALCULATED 필터 제외 안내:** 필터 시트에 "계산 필드는 정렬만 가능(저장값 없음)" 힌트 추가.
- **전역 크로스-세계관 필터 비대칭(설계 결정):** 현재 필터는 `FieldFilter.fieldId`(단일 field-def-id) 매칭이라, 전역 뷰에서 세계관 A·B의 같은 key 필드를 한 번에 못 거른다(정렬은 key로 병합). 
  - 방침: 필터 매칭을 **field-def-id → field-key 기반**으로 확장(스코프 내 같은 key의 모든 field-def-id에 OR). `FieldFilterHelper`/`FieldFilter`/저장 스키마 영향 → **가장 큰 변경**이므로 PR-F3에서 분리 가능. Global Search와 공유 로직이라 무회귀 검증 필수. 위험이 크면 별도 PR로 승격.

### F3-5. StorageAnalyzer 용량 표기
- **방침:** 저장소 화면에 "앱이 관리하는 파일 기준(OS 전체 앱 용량과 다를 수 있음)" 안내 문구 + 시스템 앱정보 링크(선택).

## PR 순서 & 검증

1. **PR-F1**(무음 유실) → 2. **PR-F2**(고지 완결) → 3. **PR-F3**(성능·경미). F3-4의 크로스-세계관 필터가 커지면 **PR-F4**로 분리.
- 각 PR: 해당 코드 재확인 → 구현 → 어드버서리얼/컴파일 리뷰(서브에이전트) → CI 그린 → master 병합.
- 데이터 정합·무결성 변경은 가능한 순수 로직을 추출해 **JVM 단위테스트** 추가(엑셀 열-present 가드, 배치 스킵 집계 등).
